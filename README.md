#
■概要：<br/>
「Haroteku」はAndroidStudioを使用してJavaで制作したゲームアプリです。<br/>
androidタブレットでのプレイを想定し、画面上から落下してくる障害物を避けながら進むシンプルな作りのゲームです。<br/>

■アプリ説明：Haroteku（言語java）：製作期間およそ3週間<br/>
本アプリはandroidの部品である「シークバー」に着目し、<br/>
「これを使えば感覚的なゲームができるのでは」という発想から開発が始まりました。<br/>
ゲームはスタート画面から「会話パート」「アクションパート」を繰り返してステージが進んでゆき、<br/>
全3ステージ＋エキストラモードで計４ステージを実装しています。<br/>

GitHubにはAndroidStudioの設定ファイルを含めず、<br/>
自身の記述したソースコードと画面設計などの資料を抜粋して掲載しています。<br/>

■苦労した点<br/>
〇ステージ切り替え処理：Stageset.java/Talkset.java<br/>
会話の終了で画面切り替え、ゲームクリア時に画面切り替えなど、<br/>
適切なタイミングで適切な画面に移行する処理に苦労しました。<br/>
全体はイメージできていても「どの処理完了の時点で画面を切り替えるべきか」を整理するのが難しく試行錯誤しながら実装しました。<br/>

〇当たり判定：Hurdle.java<br/>
当たり判定は学校の授業範囲外であり、支給された参考書にも記載がなかったため、<br/>
インターネットでの調査や、サンプルコードの検証、AIツールへの質問などを通して理解を深め、<br/>
最終的にゲームとして成立する形に仕上げることができました。<br/>

■工夫したところ<br/>
今回プログラミングをするさい理解できないことも多くあったため、<br/>
機能を細かく分解して段階的に少しずつ実装するよう意識しました。<br/>

例えば「画面切り替え」をするを実装するとき、<br/>

1.ボタンを用意する<br/>
2.ボタンを押すとテキストが切り替わるようにする<br/>
3.複数テキストを最後まで表示できるようにする<br/>
4.画面移行用のボタンを用意する<br/>
5.ボタンに画面移行の機能を付ける<br/>
6.テキスト終了時に画面移行が動くよう調整する<br/>
7.コードを書き替えて目的のページになるようにする<br/>

といったように１つ１つ動作を見ながら処理を実装しました。<br/>

それでも理解が難しい場合は、<br/>
別ファイルで処理を試してみてから本コードに移すことで、<br/>
挙動を確認しながら理解を深めました。<br/>

時間はかかりましたが、<br/>
試行錯誤を通すことで、自分で調べて理解し、実装に落とし込む力を身に着けることが出来たと感じています。<br/>
