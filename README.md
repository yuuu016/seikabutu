#
■概要：<br/>
「Haroteku」はAndroidStudioを使用してJavaで制作したゲームアプリです。<br/>
androidタブレットでのプレイを想定し、画面上から落下してくる障害物を避けながら進むシンプルな作りのゲームです。<br/>

■アプリ説明：Haroteku（言語java）：製作期間およそ3週間<br/>
本アプリはandroidの部品である「シークバー」に着目し、<br/>
「これを使えば感覚的なゲームができるのでは」という発想から開発が始まりました。<br/>
スタート画面から「会話パート」「アクションパート」を繰り返してステージが進んでゆき、<br/>
3ステージ＋エキストラモードで全４ステージを実装しています。<br/>

GitHubにはAndroidStudioの設定ファイルを含まないよう、<br/>
自分の記述したソースコードと画面設計などの資料を抜粋して掲載しています。<br/>

■苦労した点<br/>
〇ステージ切り替え処理：Stageset.java/Talkset.java<br/>
会話の終了で画面切り替え、ゲームクリア時に画面切り替え、など適切なタイミングで適切な場所に画面切り替えするという処理に苦労しました。<br/>
流れ自体はイメージできていても適切なタイミングとなると難しくて試行錯誤しながら実装しました。<br/>

〇当たり判定：Hurdle.java<br/>
当たり判定は学校で習っていないことの上、支給された参考書にもかかれていなかったことのため、<br/>
自分で調べて手探りで実装しました。検索してソースコードを移したり、AIに尋ねてみたりして形にしました。<br/>

■工夫したところ<br/>
今回プログラミングをするさい理解できないことも多くあったため、わからないことは段階的に少しずつ実装するよう工夫しました。<br/>
例えば「画面切り替え」をするを実装するとき、<br/>
「ボタンを用意する」→「ボタンを押すとテキストが切り替わる」→「複数テキストを終わりまで切り替わるようにする」<br/>
→「画面移行用のボタンを用意する」→「ボタンに画面移行の機能を付ける」→「ボタンの処理をテキスト終了時に動くようにする」<br/>
→「コードを書き替えて目的のページに移行するようにする」…など段階的に考えて目的の処理を実装しました。<br/>

今回はそれでもわからないこともあり、<br/>
その場合は別ファイルを作成してからコードを移しつつ１つ１つ動きを見ながら理解を深めていきました。<br/>
時間はかかりましたが試行錯誤の中で自分で理解して、実装に落とし込める力を身に着けることが出来ました。<br/>
